## 3. P0 当前实现说明（接手前必须了解）

在 `peak_pullback_stabilization.py` 中：

- Layer A 使用 `high_60` 的降级值：`high_60 = high_20 * 1.1`
- Layer B 使用该 `high_60` 计算回落深度
- Layer C/D 均为截面逻辑（符合当前 snapshot 接口约束）

这意味着：
- 策略可运行，但 Layer A/B 的“前高基准”不是严格 60 日高点
- P1 的核心是把“伪 high_60”替换为“真实 high_60”并验证影响

## 6. 已知风险与注意事项

- 如果 `high_60` 在 snapshot 的计算口径与 `high_20` 口径不一致，会出现 Layer A/B 漂移
- 新股或停牌导致 `high_60` 空值时，必须有明确策略（剔除或降级）
- 不要在 `filter_batch(df, target_date)` 内引入 `rolling/shift`（必须保持截面接口约束）

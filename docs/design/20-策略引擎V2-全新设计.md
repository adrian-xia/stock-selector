## 一、现有问题诊断

### 1.1 三模型评估共识

对 36 个策略使用 Claude Opus、Codex、Gemini 三个模型独立评估，取三模型综合均分排序后，发现以下核心问题：

| 问题 | 现状 | 影响 |
|------|------|------|
| **策略同质化严重** | KDJ/CCI/Williams %R 与 RSI 提供几乎相同的信息 | 冗余策略拉低信噪比，浪费计算资源 |
| **低质量策略污染** | 8 个策略三模型均分 < 5.0，信号约等于噪声 | 污染 AI 终审池，降低整体选股质量 |
| **角色定位模糊** | 基本面策略当独立选股信号用，实际应是前置过滤池 | 财务安全 ≠ 有投资价值，用法错位 |
| **扁平权重** | 所有策略地位平等，仅靠命中率事后加权 | 低质量策略与高质量策略"同票同权" |
| **无市场状态感知** | 牛市/熊市/震荡市用同一套策略和参数 | 趋势策略在震荡市假信号泛滥 |
| **Pipeline 结构僵化** | 技术→基本面的串行过滤，OR 逻辑合并 | 无法实现"基本面选池+技术面触发"的最优范式 |

## 二、全新架构设计

### 2.1 核心设计理念

三模型共识提炼出一个关键范式：

> **基本面选池（Universe），技术面扣扳机（Trigger）**

- 基本面因子的价值是"防踩雷+划定优质底仓池"，不是触发买卖信号
- 技术面因子的价值是"捕捉资金异动和情绪拐点"，是真正的入场信号
- 策略不再"平等投票"，而是分角色协作

### 2.2 策略分层架构

```
┌─────────────────────────────────────────────────────────┐
│                   新 Pipeline 架构                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  Layer 0 ── 硬性排除（SQL）                               │
│  ├─ ST/退市/停牌/上市不足60天                              │
│  ├─ 当日成交额 < 500万 或 换手率 < 0.1%                    │
│  └─ 涨跌停（已触板的无法次日买入）                           │
│                                                         │
│  Layer 1 ── 质量底池（基本面过滤）                          │
│  ├─ Guard: 财务安全 + 现金流质量（排雷硬门槛）               │
│  └─ Scorer: 综合质量评分（行业中性化 Z-Score）               │
│  → 输出：Quality Universe（约全市场 Top 40%）              │
│                                                         │
│  Layer 2 ── 信号触发（技术面策略）                          │
│  ├─ 进攻组：龙回头/首阴反包/放量突破/后量超前量              │
│  ├─ 趋势组：缩量回调/高位回落企稳/回调半分位/ATR突破          │
│  └─ 底部组：地量见底/量缩价稳/缩量上涨                      │
│  → 输出：每只股票的触发信号列表 + 信号置信度                  │
│                                                         │
│  Layer 3 ── 多因子融合排序                                 │
│  ├─ 信号强度分（策略静态权重 × 市场状态系数）                 │
│  ├─ 质量底分（Layer 1 的综合质量得分）                       │
│  ├─ 动态加权（基于滚动命中率/盈亏比自适应）                   │
│  └─ 风格增益（成长池/价值池/红利池分别加分）                   │
│  → 输出：Top N 排序结果                                    │
│                                                         │
│  Layer 4 ── AI 终审（可选）                                │
│  └─ Gemini Flash 综合分析，补充新闻/舆情/板块维度             │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 2.3 策略角色重新定义

不再使用 `category = "technical" | "fundamental"` 的二分法，改为 **5 种角色**：

| 角色 | 英文标识 | 说明 | 执行位置 |
|------|---------|------|---------|
| **排雷卫兵** | `guard` | 硬性排除条件，不通过则直接剔除 | Layer 1 |
| **质量评分** | `scorer` | 输出连续分数（0-100），用于排序加权 | Layer 1 |
| **风格标签** | `tagger` | 为股票打标签（成长/价值/红利），用于风格池 | Layer 1 |
| **信号触发** | `trigger` | 产出 True/False 买入信号，真正的选股核心 | Layer 2 |
| **辅助确认** | `confirmer` | 不独立触发，为已触发信号提供加分/减分 | Layer 3 |

## 三、Pipeline 执行流程详细设计

### 3.1 Layer 0: 硬性排除（SQL）

与现有 `_layer1_base_filter` 基本一致，增加以下条件：

```sql
-- 新增排除条件
AND sd.amount >= 500        -- 当日成交额 >= 500万（排除僵尸股）
AND sd.pct_chg > -9.9       -- 非跌停（无法次日买入）
AND sd.pct_chg < 9.9         -- 非涨停（溢价已充分）
-- 上市时间过滤移到这层
AND s.list_date <= :min_list_date  -- 上市满60天
```

### 3.2 Layer 1: 质量底池

**关键变化**：基本面从"串行过滤器"变为"并行评分器"。

```python
class Layer1Result:
    """Layer 1 输出结构。"""
    ts_code: str
    passed_guard: bool          # 是否通过排雷关卡
    quality_score: float        # 综合质量评分（0-100）
    tags: list[str]             # 风格标签：["growth", "dividend", "value"]
```

执行逻辑：
1. **Guard 策略**以 AND 逻辑执行 — 任一 guard 不通过则剔除
2. **Scorer 策略**输出连续分数（不做硬过滤），存入 `quality_score`
3. **Tagger 策略**为通过的股票打风格标签

输出：所有通过 guard 的股票（约全市场 40%），每只附带 quality_score + tags。

### 3.3 Layer 2: 信号触发

**关键变化**：策略不再 OR 合并为一个布尔，而是输出独立信号列表。

```python
class SignalResult:
    """单个信号触发结果。"""
    strategy_name: str          # 触发策略名
    signal_group: str           # "aggressive" | "trend" | "bottom"
    confidence: float           # 信号置信度 0.0-1.0（策略内部计算）
    static_weight: float        # 策略静态权重（基于三模型均分）
```

每只股票可以被多个 trigger 同时命中，输出信号列表而非布尔。

### 3.4 Layer 3: 多因子融合排序

**关键变化**：引入市场状态感知 + 滚动绩效自适应。

```
最终得分 = 信号强度分 × 0.5 + 质量底分 × 0.2 + 动态加权 × 0.2 + 风格增益 × 0.1

其中：
  信号强度分 = Σ(策略静态权重 × 市场状态系数 × 信号置信度 × 确认加分)
  质量底分   = Layer 1 综合质量评分 / 100
  动态加权   = 基于滚动30天命中率+盈亏比的自适应乘数
  风格增益   = 震荡市 dividend +0.3 / 趋势市 growth +0.3 / 普通 0
```

```python
class MarketRegime(Enum):

def detect_market_regime(index_data: pd.DataFrame) -> MarketRegime:
    """基于上证指数 + 成交额判断市场状态。

    规则：
    - 指数 > MA20 > MA60 且 MA20 斜率 > 0 → BULL
    - 指数 < MA20 < MA60 且 MA20 斜率 < 0 → BEAR
    - 其他 → RANGE
    """
```

### 3.5 Layer 4: AI 终审

与现有 Layer 5 相同，调用 Gemini Flash 进行综合分析。传入新增信息：
- 市场状态 (MarketRegime)
- 信号组分类和置信度
- 质量评分和风格标签

## 四、BaseStrategy 重设计

### 4.1 新基类

```python
from abc import ABC, abstractmethod
from dataclasses import dataclass, field
from datetime import date

from typing import Literal

import pandas as pd


class StrategyRole(str, Enum):
    """策略角色。"""
    GUARD = "guard"             # 排雷卫兵：硬性排除
    SCORER = "scorer"           # 质量评分：输出连续分数

    """信号分组（仅 trigger 角色使用）。"""
    AGGRESSIVE = "aggressive"   # 进攻组
    TREND = "trend"             # 趋势组
    BOTTOM = "bottom"           # 底部组


@dataclass
class StrategySignal:
    """策略输出的信号。"""
    ts_code: str
    confidence: float = 1.0     # 信号置信度 0.0-1.0


class BaseStrategy(ABC):
    """策略基类（V2 重设计）。"""

    name: str = ""
    display_name: str = ""
    role: StrategyRole = StrategyRole.TRIGGER
    signal_group: SignalGroup | None = None  # 仅 trigger 有
    description: str = ""
    default_params: dict = {}
    # 三模型综合均分（用于静态权重计算）
    ai_rating: float = 5.0

    def __init__(self, params: dict | None = None) -> None:
        self.params = {**self.default_params}
        if params:
            self.params.update(params)

    @abstractmethod
    async def execute(
        self,
        df: pd.DataFrame,
        target_date: date,
    ) -> list[StrategySignal] | pd.Series | float:
        """执行策略。

        根据角色不同，返回值不同：
        - guard: pd.Series[bool]（True=通过）
        - scorer: pd.Series[float]（0-100 连续分数）
        - tagger: pd.Series[bool]（True=打上标签）
        - trigger: list[StrategySignal]（命中信号列表，含置信度）
        - confirmer: pd.Series[float]（加分系数 0.0-1.0）
        """
        ...

    @property
    def static_weight(self) -> float:
        """基于三模型均分的静态权重（归一化到 0-1）。"""
        return self.ai_rating / 8.32  # 除以最高分归一化
```

## 七、优化目标升级

### 7.1 现有优化目标

当前 MarketOptimizer 仅优化"5 日命中率"（`max_5d_return >= 3%`）。

### 7.2 新优化目标：三指标联合

```python
@dataclass
class OptimizationTarget:
    """优化目标（三指标联合）。"""
    hit_rate: float         # 5日命中率（权重 0.4）
    profit_loss_ratio: float  # 盈亏比（权重 0.35）
    max_drawdown: float       # 最大回撤（权重 0.25）

    @property
    def composite_score(self) -> float:
        """综合评分 = 命中率×0.4 + 盈亏比×0.35 - 最大回撤×0.25"""
        return (
            self.hit_rate * 0.4
            + self.profit_loss_ratio * 0.35
            - self.max_drawdown * 0.25
        )
```

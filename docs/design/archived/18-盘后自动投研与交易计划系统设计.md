# 盘后自动投研与交易计划系统设计（StarMap v1）

> [!WARNING]
> SUPERSEDED: 本文档为概览版，已被 `18-盘后自动投研与交易计划系统设计-详细版.md` 取代。
> 实施、评审与验收请仅以详细版为准。

> 面向现有 `stock-selector` 系统的增强设计文档
> 目标：将结构化量化数据（Tushare）与非结构化信息（新闻+LLM）融合，形成可执行的次日交易计划

---

## 1. 背景与目标

当前系统已具备：
- 完整的盘后数据同步链路（行情、技术指标、部分资金与基础面）
- 策略工厂与可配置策略执行框架
- 盘后报告与通知机制（含 Telegram）

需要新增能力：
1. 宏观新闻 -> 行业主题 -> 个股机会的自上而下研判
2. 盘后自动输出“可执行交易计划”（入场/止损/止盈/仓位）
3. 用风险分层机制控制策略激进程度，避免“只看信号不看环境”

---

## 2. 系统定位

StarMap 是现有系统上的“投研决策层”，不是替代原策略层。

- 原策略层负责：高效、可回测的规则筛选
- StarMap 负责：跨数据源语义融合与交易计划生成
- 交付形态：结构化报告 + 交易计划卡片 + 风险提示

---

## 3. 总体架构（Top-Down 漏斗）

### 3.1 层次划分

1. **宏观事件层（LLM）**
- 输入：盘中/盘后财经新闻、政策、海外市场异动
- 输出：风险偏好、利好行业、利空行业、摘要因果链

2. **市场状态层（量化）**
- 输入：宽基指数、涨跌停结构、连板高度、市场成交结构
- 输出：市场温度、风格偏好、总仓位建议上限

3. **行业共振层（量化+语义）**
- 输入：行业涨跌、行业资金、LLM 行业标签
- 输出：可交易行业池（消息和资金同向）

4. **个股执行层（策略）**
- 输入：现有策略结果 + 个股资金与技术状态
- 输出：交易计划（Entry/Stop/TP/Position）

### 3.2 数据流

`新闻抓取 -> LLM结构化 -> 市场宽指评分 -> 行业共振评分 -> 个股策略筛选 -> 交易计划生成 -> 推送`

---

## 4. 功能模块设计

## 4.1 宏观新闻分析模块（新增）

### 输入
- 财经快讯与新闻源（财联社、华尔街见闻、公告摘要等）

### 处理
- 清洗去重
- 按事件类型聚类（政策/地缘/流动性/产业）
- LLM 提取结构化 JSON

### 输出 JSON 规范

```json
{
  "trade_date": "2026-03-02",
  "global_risk_appetite": "high|mid|low",
  "positive_sectors": ["军工", "黄金"],
  "negative_sectors": ["新能源"],
  "macro_summary": "100字以内摘要",
  "key_drivers": ["事件A->影响B->传导C"]
}
```

---

## 4.2 市场状态评分模块（新增）

### 核心指标
- 指数位置：`close vs MA20/MA60`
- 情绪指标：涨停数、跌停数、连板高度、昨日涨停溢价
- 成交特征：量能扩张/萎缩、风格偏向（大盘/小盘）

### 输出
- `market_regime`: bull / range / bear
- `risk_score`: 0~100
- `position_cap`: 0.2~1.0（建议总仓位上限）

---

## 4.3 行业共振模块（新增）

### 目标
只保留“消息面 + 资金面 + 价格面”至少双重确认的行业。

### 行业得分（示例）

`sector_score = 0.35*news_score + 0.35*moneyflow_score + 0.30*trend_score`

### 输出
- `focus_sectors`: 次日重点行业列表
- 每个行业附带：触发原因、风险点、置信度

---

## 4.4 个股交易计划模块（新增）

### 输入
- 现有策略命中结果（如 volume-price-pattern、peak-pullback-stabilization）
- 行业共振结果
- 个股技术/资金指标

### 交易计划模板

```json
{
  "ts_code": "300871.SZ",
  "plan_type": "breakout|pullback",
  "entry_rule": "高开不超3%，30分钟不破昨收可试仓",
  "stop_loss_rule": "跌破20日线且放量，执行止损",
  "take_profit_rule": "前高附近分批止盈",
  "position_suggestion": 0.15,
  "reasoning": ["行业共振", "量价结构", "市场风险可控"]
}
```

## 5. 与现有系统集成方案

## 5.1 新增目录建议

- `app/research/news/`：新闻抓取与清洗
- `app/research/llm/`：Prompt 与结构化输出解析
- `app/research/scoring/`：市场与行业评分
- `app/research/trade_plan/`：交易计划生成器
- `app/api/research.py`：查询 StarMap 结果接口

## 5.2 调度集成

在现有 `run_post_market_chain` 后半段增加：
1. 新闻抓取与 LLM 分析
2. 市场与行业评分
3. 交易计划生成
4. 报告组装与推送

失败策略：
- 新闻/LLM 失败不阻断主链路，回退为“纯量化模式”
- 计划生成失败时保留策略结果并附加降级提示

## 7. 报告输出规范

每日报告结构：
1. 宏观与市场一句话结论
2. 风险等级与总仓位建议
3. 重点行业 TopN（含理由）
4. 个股交易计划 TopN（可执行条件）
5. 当日失败项与降级项（透明化）

输出形态：
- Markdown 主报告
- JSON 机器可读结果（便于后续前端展示）

## 8. 开发里程碑

### M1（1~2天）
- 打通新闻抓取与 LLM 结构化输出
- 落表 `macro_signal_daily`

### M2（2~3天）
- 市场/行业评分模块
- 落表 `sector_resonance_daily`

### M3（2~3天）
- 交易计划生成器
- 落表 `trade_plan_daily` + 报告联动

### M4（1~2天）
- 回测与复盘验证（命中率、盈亏比、回撤）
- 参数校准与推送策略优化

---

## 9. 风险与约束

- LLM 可能产生不稳定输出：必须强制 JSON schema 校验
- 新闻噪声高：需要去重与可信源白名单
- 不可将 LLM 直接作为买卖信号：仅作为“解释层+筛选增强”
- 交易纪律需硬编码：止损规则优先级高于叙事逻辑

---

## 10. 验收标准

1. 连续 10 个交易日稳定生成报告
2. 失败降级路径可用（无 silent failure）
3. 报告中每只个股都有明确 Entry/Stop/TP
4. 复盘指标可追溯（能回看每日评分与决策依据）

---

## 11. 与当前策略体系的衔接建议

- 保留现有策略评分主导地位
- StarMap 提供“行业与宏观加权因子”
- 对 `peak-pullback-stabilization` 这类形态策略：
  - 若所属行业在 `focus_sectors`，可提高排序权重
  - 若市场风险分数过低，自动降低建议仓位

---

## 12. 结语

该方案不会推翻现有系统，而是在现有高可用盘后链路上新增“投研语义层”和“计划执行层”。
最终目标是：
- 盘后形成结构化计划
- 盘中减少主观冲动
- 让策略、资金、叙事三者同向时再出手

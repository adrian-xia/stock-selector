# StarMap Phase 0: 新闻源 PoC 结论

> 日期：2026-03-05
> 状态：✅ 已完成

## 评估结果

| 数据源 | 可达 | 条数 | 宏观覆盖率 | 延迟 | 结论 |
|--------|------|------|-----------|------|------|
| **新浪 7x24 快讯** | ✅ | 60 条/3 页 | **60%** | 135ms | ✅ **推荐主数据源** |
| Tushare news/major_news | ❌ | 0 | 0% | 91ms | 非交易日返回空，或需高权限 |
| 财联社电报 | ❌ | 0 | 0% | 268ms | API 404（endpoint 已变更） |

## 结论

### 推荐方案：新浪 7x24 快讯作为 StarMap V1 主数据源

**理由：**
1. **已验证可用**：现有 `app/data/sources/sina.py` 的 `SinaCrawler` 已跑通
2. **宏观覆盖率高**：60% 的快讯与宏观/行业/市场相关（政府工作报告、央行政策、指数信息等）
3. **零额外成本**：无需付费 API，无需额外 token/积分
4. **实时性好**：快讯流接近实时更新，135ms 延迟

**样例宏观新闻：**
- 政府工作报告：制定实施城乡居民增收计划...
- 设立1000亿元财政金融协同促内需专项资金...
- 沪深两市成交额超5000亿元...
- 安排超长期特别国债2500亿元支持消费品以旧换新

### StarMap Scope：**Full（完整模式）**

PoC 通过，Phase 1 可按设计文档完整实施，包括：
- 新闻抓取 → LLM 结构化 → 行业对齐
- 市场评分 + 行业共振
- 交易计划增强

### 实现路径

1. **改造 `SinaCrawler`**：复用现有类，新增 `fetch_macro()` 方法——不做 per-stock 匹配，直接返回全量快讯
2. **新增 `app/research/news/fetcher.py`**：抽象 `NewsSource` 接口，Sina 为首个实现
3. **去重**：`dedupe.py` 使用 Jaccard 分词相似度（阈值 0.7）
4. **Tushare 保留观察**：后续交易日再测试 `news`/`major_news`；若可用则作为补充源
5. **CLS 不作为 V1 依赖**：API 不稳定，等官方更新后再评估

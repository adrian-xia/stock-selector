# 15-策略设计：高位回落企稳二次启动（Peak Pullback Stabilization）

> **版本：** v1.1
> **日期：** 2026-03-01
> **状态：** PRD（待实施）
> **变更：** v1.1 修复 Gemini 审查发现的三个逻辑漏洞（时间维度坍塌、振幅涨幅互斥、Layer A/B 参数冲突）及一个企稳判断隐患
> **关联：** 05-详细设计-量价综合选股策略.md、14-系统设计-V4量价配合策略优化任务.md

## 1. 背景与目标

### 1.1 问题来源

在实盘观察中，金风科技、上大股份等个股呈现出一种典型走势：

1. **前期主升浪**：有明显的趋势性上涨，说明主力资金参与
2. **高位见顶回落**：价格从高点回落 10-30%，但成交量同步萎缩（非恐慌性抛售）
3. **缩量企稳**：在某一支撑位横盘整理 5-15 个交易日，量能持续萎缩
4. **小阳启动**：近期出现小阳线，量能温和放大，二次启动信号出现

这种形态与 V4 量价配合（龙回头）的核心区别在于：

| 对比项 | V4 龙回头 | 本策略（高位回落企稳） |
|--------|-----------|----------------------|
| 触发点 | 放量突破新高后回踩 | 前期高点回落后企稳 |
| 回落幅度 | 通常 5-15%（浅回踩） | 通常 10-30%（深度调整） |
| 企稳位置 | 突破点附近 / MA 支撑 | 前期平台 / 关键均线 |
| 状态追踪 | 需要观察池（多日） | 单日快照可判断 |
| 适用场景 | 强势股短期回踩 | 调整充分后的二次启动 |

### 1.2 目标

新增 `peak-pullback-stabilization`（高位回落企稳）策略，捕捉经历充分调整后重新启动的个股，与 V4 形成互补覆盖。

## 2. 策略定义

### 2.1 策略标识

| 字段 | 值 |
|------|----|
| `name` | `peak-pullback-stabilization` |
| `display_name` | `高位回落企稳` |
| `category` | `technical` |
| `description` | 前期主升浪后高位回落，缩量企稳后小阳二次启动 |

### 2.2 信号逻辑

策略分为四个判断层，全部满足才产生信号：

```
Layer A：前期有主升浪（确认主力参与）
    ↓
Layer B：经历高位回落（确认调整充分）
    ↓
Layer C：缩量企稳（确认抛压耗尽）
    ↓
Layer D：小阳启动信号（确认二次启动）
```

### 2.3 各层详细条件

#### Layer A：前期主升浪确认

| # | 条件 | 字段 | 默认阈值 | 说明 |
|---|------|------|---------|------|
| A1 | 60日高点显著高于长期均线 | `high_60 > ma60 * (1 + min_peak_rise_pct/100)` | 20% | 证明前期有过主升浪，高点乖离 MA60 超 20% |
| A2 | 当前价格仍在 MA60 附近 | `close >= ma60 * (1 - ma_tolerance)` | 容忍 3% | 大趋势未破坏 |
| A3 | 正常交易 | `vol > 0` | — | 排除停牌 |

> **v1.1 修正（漏洞三）：** 原版 Layer A 用 `pullback_pct >= min_peak_rise_pct` 与 Layer B 判断同一变量，导致 Layer B 的 `min_pullback_pct` 参数网格（8/10/12/15%）全部被 Layer A 的 20% 下限覆盖失效。修正后 Layer A 改为 `high_60 > ma60 * 1.2`，语义变为"高点曾经显著高于长期均线"，与 Layer B 的回落幅度判断完全解耦。

#### Layer B：高位回落确认

| # | 条件 | 字段 | 默认阈值 | 说明 |
|---|------|------|---------|------|
| B1 | 当前价距近期高点回落幅度 | `(high_60 - close) / high_60` | 10% ~ 35% | 回落不能太浅（未调整）也不能太深（趋势破坏） |
| B2 | 当前价仍在 MA60 上方或附近 | `close >= ma60 * (1 - ma_tolerance)` | `ma_tolerance=0.03` | 长期趋势未破坏 |
| B3 | 当前价低于 MA20 | `close < ma20` | — | 确认经历了调整，非高位横盘 |

#### Layer C：缩量企稳确认（过去 N 天）

| # | 条件 | 字段 | 默认阈值 | 说明 |
|---|------|------|---------|------|
| C1 | 过去 5 日平均量比萎缩 | `vol_ma5 / vol_ma10 < max_vol_ratio` | 0.8 | 近期整体处于缩量状态 |
| C2 | 昨日振幅小（企稳期 K 线） | `prev_amplitude < max_amplitude` | 3.0% | 昨天价格稳定，非今天启动日 |
| C3 | 短期止跌（站上 MA5） | `close >= ma5` | — | 排除持续阴跌，确认短期止跌 |
| C4 | 非跌停 | `pct_chg > -9.5` | — | 排除恐慌性下跌 |

> **v1.1 修正（漏洞一 + 漏洞二 + 隐患四）：**
> - 原版 Layer C 和 Layer D 都在判断今天，导致量比被夹在 0.6-0.8 的死区，实际是"缩量微涨"而非"放量启动"。
> - 修正后 Layer C 改为判断**过去几天**的状态：用 `vol_ma5/vol_ma10` 衡量近期整体缩量，用**昨日振幅**（`prev_amplitude`）而非今日振幅约束企稳期 K 线，彻底解除与 Layer D 的互斥。
> - 新增 `close >= ma5` 排除持续阴跌（隐患四修正）。
> - `prev_amplitude` = `(high_prev - low_prev) / close_prev * 100`，用 Pipeline 已有的 `high`/`low`/`close` 的前日字段计算，或在 Pipeline 中补充 `amplitude_prev`。

#### Layer D：今日放量小阳启动

| # | 条件 | 字段 | 默认阈值 | 说明 |
|---|------|------|---------|------|
| D1 | 当日收阳 | `close > open` | — | 多头占优 |
| D2 | 涨幅在合理范围 | `min_pct_chg ≤ pct_chg ≤ max_pct_chg` | 0.5% ~ 7.0% | 小阳到中阳，非涨停追高 |
| D3 | 量能放大（相对近期缩量期） | `vol_ratio >= min_signal_vol_ratio` | 1.0 | 启动日量比需 ≥ 1.0，有效放量 |

> **v1.1 修正（漏洞一）：** `min_signal_vol_ratio` 从 0.6 提升至 1.0，启动日必须有效放量（量比 ≥ 均量），而非原来的缩量微涨。`max_pct_chg` 从 5.0% 放宽至 7.0%，避免与振幅约束产生死区。

## 3. 参数设计

### 3.1 默认参数

```python
DEFAULT_PARAMS = {
    # Layer A：前期主升浪
    "min_peak_rise_pct": 20.0,      # 高点距 MA60 的最小乖离率 %（证明有过主升浪）
    "ma_tolerance": 0.03,           # MA60 容忍偏差（允许当前价略低于 MA60）

    # Layer B：高位回落
    "min_pullback_pct": 10.0,       # 最小回落幅度 %（距 60 日高点）
    "max_pullback_pct": 35.0,       # 最大回落幅度 %（距 60 日高点）

    # Layer C：缩量企稳（过去几天）
    "max_vol_ratio": 0.8,           # 近期 vol_ma5/vol_ma10 上限（整体缩量）
    "max_amplitude": 3.0,           # 昨日振幅上限 %（企稳期 K 线）

    # Layer D：今日放量启动
    "min_pct_chg": 0.5,             # 启动日最小涨幅 %
    "max_pct_chg": 7.0,             # 启动日最大涨幅 %（排除涨停追高）
    "min_signal_vol_ratio": 1.0,    # 启动日最小量比（必须有效放量）
}
```

### 3.2 参数空间（用于优化）

| 参数 | 范围 | 组合数 | 说明 |
|------|------|--------|------|
| `min_pullback_pct` | 8, 10, 12, 15 | 4 | 回落下限 |
| `max_pullback_pct` | 25, 30, 35, 40 | 4 | 回落上限 |
| `max_vol_ratio` | 0.6, 0.7, 0.8, 0.9 | 4 | 企稳期缩量程度 |
| `min_pct_chg` | 0.3, 0.5, 1.0 | 3 | 启动日最小涨幅 |
| `max_amplitude` | 2.0, 3.0, 4.0 | 3 | 企稳期昨日振幅上限 |
| `min_signal_vol_ratio` | 0.8, 1.0, 1.2 | 3 | 启动日放量要求 |

**总组合数：** 4 × 4 × 4 × 3 × 3 × 3 = **1728 组**（仍适合全市场优化器）

## 4. 数据依赖

### 4.1 所需字段

| 字段 | 来源 | 状态 | 说明 |
|------|------|------|------|
| `close` / `open` / `high` / `low` | `stock_daily` | ✅ 已有 | 今日 OHLC |
| `vol` / `pct_chg` | `stock_daily` | ✅ 已有 | 成交量、涨跌幅 |
| `pct_chg_prev` | Pipeline 前日快照 | ✅ 已有 | 昨日涨跌幅（近似企稳判断） |
| `close_prev` | Pipeline 前日快照 | ✅ 已有 | 昨日收盘价 |
| `vol_ratio` | `technical_daily` | ✅ 已有 | 今日量比 |
| `vol_ma5` / `vol_ma10` | `technical_daily` | ✅ 已有 | 5/10 日均量（用于近期缩量判断） |
| `ma5` / `ma20` / `ma60` | `technical_daily` | ✅ 已有 | 均线 |
| `high_60` | Pipeline 预计算 | ⚠️ 待补充 | 60 日最高价，暂用 `high_20 * 1.1` 降级 |
| `high_prev` / `low_prev` | Pipeline 前日快照 | ⚠️ 待补充 | 昨日高低价，用于精确计算昨日振幅 |

### 4.2 high_60 字段

`high_60` 是 Pipeline 在构建市场快照时预计算的字段（类似 `high_20`）。

**检查是否已存在：**

```python
# pipeline.py 中检查
if "high_60" not in df.columns:
    # 降级方案：用 high_20 * 1.1 近似（不推荐，应补充预计算）
    df["high_60"] = df.get("high_20", df["close"]) * 1.1
```

**推荐在 Pipeline 中补充预计算：**

```sql
-- technical_daily 或 Pipeline snapshot 中添加
SELECT ts_code, MAX(high) OVER (
    PARTITION BY ts_code
    ORDER BY trade_date
    ROWS BETWEEN 59 PRECEDING AND CURRENT ROW
) AS high_60
FROM stock_daily
```

### 5.2 实现框架（v1.1 修正版）

```python
"""高位回落企稳策略（v1.1）。

逻辑：前期主升浪后高位回落，近期缩量企稳，今日放量小阳二次启动。
核心修正：Layer C 判断过去几天，Layer D 判断今天，时间维度分离。
"""

from datetime import date
import pandas as pd
from app.strategy.base import BaseStrategy


class PeakPullbackStabilizationStrategy(BaseStrategy):
    """高位回落企稳策略：主升浪后深度调整，缩量企稳后放量小阳二次启动。"""

    name = "peak-pullback-stabilization"
    display_name = "高位回落企稳"
    category = "technical"
    description = "前期主升浪后高位回落，缩量企稳后放量小阳二次启动"
    default_params = {
        "min_peak_rise_pct": 20.0,
        "ma_tolerance": 0.03,
        "min_pullback_pct": 10.0,
        "max_pullback_pct": 35.0,
        "max_vol_ratio": 0.8,
        "max_amplitude": 3.0,
        "min_pct_chg": 0.5,
        "max_pct_chg": 7.0,
        "min_signal_vol_ratio": 1.0,
    }

    async def filter_batch(self, df: pd.DataFrame, target_date: date) -> pd.Series:
        p = self.params

        # ── 今日字段 ──────────────────────────────────────────────
        close     = df.get("close",     pd.Series(dtype=float)).astype(float).fillna(0)
        open_     = df.get("open",      pd.Series(dtype=float)).astype(float).fillna(0)
        high      = df.get("high",      pd.Series(dtype=float)).astype(float).fillna(0)
        low       = df.get("low",       pd.Series(dtype=float)).astype(float).fillna(0)
        vol       = df.get("vol",       pd.Series(dtype=float)).astype(float).fillna(0)
        pct_chg   = df.get("pct_chg",   pd.Series(dtype=float)).astype(float).fillna(0)
        vol_ratio = df.get("vol_ratio", pd.Series(dtype=float)).astype(float).fillna(0)
        ma5       = df.get("ma5",       pd.Series(dtype=float)).astype(float).fillna(0)
        ma20      = df.get("ma20",      pd.Series(dtype=float)).astype(float).fillna(0)
        ma60      = df.get("ma60",      pd.Series(dtype=float)).astype(float).fillna(0)
        vol_ma5   = df.get("vol_ma5",   pd.Series(dtype=float)).astype(float).fillna(0)
        vol_ma10  = df.get("vol_ma10",  pd.Series(dtype=float)).astype(float).fillna(0)

        # ── 前日字段（用于企稳期振幅判断）────────────────────────
        close_prev = df.get("close_prev", pd.Series(dtype=float)).astype(float).fillna(0)
        # prev_amplitude = (昨高 - 昨低) / 昨收 * 100
        # Pipeline 暂无 high_prev/low_prev，用 pct_chg_prev 近似：
        # 若昨日涨幅绝对值 < max_amplitude，视为企稳（保守近似）
        pct_chg_prev = df.get("pct_chg_prev", pd.Series(dtype=float)).astype(float).fillna(0)

        # ── high_60：60 日最高价 ──────────────────────────────────
        if "high_60" in df.columns:
            high_60 = df["high_60"].astype(float).fillna(0)
        elif "high_20" in df.columns:
            high_60 = df["high_20"].astype(float).fillna(0) * 1.1
        else:
            high_60 = close * 1.25

        # 回落幅度 = (60日高点 - 今收) / 60日高点
        pullback_pct = (high_60 - close) / high_60.replace(0, float("nan")) * 100
        pullback_pct = pullback_pct.fillna(0)

        # ── Layer A：前期主升浪（高点显著高于 MA60）──────────────
        layer_a = (
            (high_60 > 0) &
            (high_60 > ma60 * (1 + p["min_peak_rise_pct"] / 100.0)) &
            (close >= ma60 * (1 - p["ma_tolerance"])) &
            (vol > 0)
        )

        # ── Layer B：高位回落确认 ─────────────────────────────────
        layer_b = (
            (pullback_pct >= p["min_pullback_pct"]) &
            (pullback_pct <= p["max_pullback_pct"]) &
            (close < ma20)  # 已跌破 MA20，确认经历调整
        )

        # ── Layer C：近期缩量企稳（过去几天，非今天）────────────
        # C1: 近期整体缩量（5日均量 / 10日均量 < 阈值）
        vol_ratio_recent = vol_ma5 / vol_ma10.replace(0, float("nan"))
        vol_ratio_recent = vol_ratio_recent.fillna(1.0)
        # C2: 昨日涨跌幅绝对值小（企稳期 K 线平稳）
        prev_stable = pct_chg_prev.abs() < p["max_amplitude"]
        # C3: 短期止跌（站上 MA5，排除持续阴跌）
        above_ma5 = (ma5 > 0) & (close >= ma5)

        layer_c = (
            (vol_ratio_recent < p["max_vol_ratio"]) &
            prev_stable &
            above_ma5 &
            (pct_chg > -9.5)
        )

        # ── Layer D：今日放量小阳启动 ─────────────────────────────
        layer_d = (
            (close > open_) &
            (pct_chg >= p["min_pct_chg"]) &
            (pct_chg <= p["max_pct_chg"]) &
            (vol_ratio >= p["min_signal_vol_ratio"])  # 今日量比 >= 1.0，有效放量
        )

        return layer_a & layer_b & layer_c & layer_d
```

> **数据依赖说明：**
> - `vol_ma5` / `vol_ma10`：Pipeline 已有，用于 Layer C 近期缩量判断
> - `pct_chg_prev`：Pipeline 已有，用于近似昨日企稳判断
> - `high_60`：Pipeline 暂无，降级用 `high_20 * 1.1`；建议 P1 阶段补充预计算
> - `high_prev` / `low_prev`：暂无，P1 阶段补充后可精确计算昨日振幅

## 7. 胜率预估与风险

### 7.1 理论胜率分析

| 维度 | 分析 |
|------|------|
| 历史形态胜率 | 5 日胜率预估 **55-65%**，10 日胜率 **50-60%** |
| 盈亏比 | 预估 **1.5-2.5**（回落充分，反弹空间相对明确） |
| 适用市场 | 震荡市、结构性行情效果最佳；单边熊市慎用 |
| 信号频率 | 预估每日 **5-20 只**（比 V4 更多，因为无需多日追踪） |

### 7.2 主要风险

| 风险 | 描述 | 缓解措施 |
|------|------|---------|
| 趋势破坏 | 回落超过 35% 可能是趋势反转 | `max_pullback_pct` 限制 |
| 假突破 | 小阳后继续下跌 | 结合大盘环境过滤（可选扩展） |
| 基本面恶化 | 技术面企稳但基本面变差 | 可叠加基本面过滤（扩展方向） |
| 流动性不足 | 小盘股缩量企稳后无法成交 | 可增加最小成交额过滤 |

### 7.3 与 V4 的协同

- **互补关系**：V4 捕捉浅回踩（5-15%），本策略捕捉深调整（10-35%）
- **信号叠加**：同一只股票同时触发两个策略，可作为强信号
- **报告展示**：在盘后报告中单独列出，标注 `🔄 二次启动`

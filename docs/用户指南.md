## 一、先看懂系统现在怎么用

### 1. 你在前端真正会接触到的策略

- **工作台可勾选的**：只有 **10 个 Trigger 策略**
- **策略配置页可开关和调参数的**：也只有 **10 个 Trigger 策略**
- **系统自动运行但不单独勾选的**：
  - 2 个 Guard：负责排雷
  - 1 个 Scorer：负责给股票打 0-100 质量分
  - 2 个 Tagger：负责打成长/红利标签并给风格增益
  - 5 个 Confirmer：负责给已触发的信号加分

### 2. 当前主流程不是“命中几个策略就排前面”

V2 的核心逻辑是：

1. **Layer 0**：先做 SQL 硬过滤  
   去掉 ST、停牌、退市、低流动性、涨跌停、上市不足一年等股票。
2. **Layer 1**：质量底池  
   先过 2 个 Guard，再给每只股票打 `quality_score`，同时生成 `growth` / `dividend` 风格标签。
3. **Layer 2**：Trigger 触发  
   只跑你勾选的 Trigger；命中任意一个 Trigger 的股票进入候选池。
4. **Layer 3**：多因子融合排序  
   结合市场状态、静态权重、滚动绩效、确认加分、质量底分、风格增益做最终排序。

### 3. 当前系统里的策略总数

| 类别 | 数量 | 是否在工作台直接勾选 | 说明 |
|---|---:|---|---|
| Trigger | 10 | 是 | 你主动选择的信号入口 |
| Guard | 2 | 否 | 自动排雷 |
| Scorer | 1 | 否 | 自动给质量分 |
| Tagger | 2 | 否 | 自动给成长/红利标签 |
| Confirmer | 5 | 否 | 自动按信号组加分 |
| 独立 V4 | 1 | 否（不在 V2 工作台列表） | 遗留但仍保留 |

## 二、快速上手

### 1. 启动系统

如果你是本地运行，最短路径如下：

```bash
# 后端
uv run uvicorn app.main:app --reload

# 前端
cd web
pnpm install
pnpm dev
```

如果数据库还是空的，先做一次初始化：

```bash
uv run python -m scripts.init_data
```

### 2. 打开页面

- 前端工作台：`http://localhost:5173`
- 后端 Swagger：`http://localhost:8000/docs`
- 健康检查：`http://localhost:8000/health`

### 3. 日常使用顺序

建议按这个顺序使用：

1. 在**工作台**选择 2-4 个 Trigger
2. 选定 `target_date`
3. 如有需要，加上 `industries` / `markets` 过滤
4. 点击执行，查看 `market_regime`、`quality_score`、`confirmed_bonus`、`final_score`
5. 如果某个 Trigger 太宽或太严，再去**策略配置页**调默认参数
6. 对稳定使用的 Trigger，再去做**回测**或**参数优化**

### 4. 两个重要事实

- **工作台是 Trigger 驱动**：你勾选的是“触发条件”，不是整套评分器
- **策略配置页只改 Trigger 参数**：Guard / Scorer / Tagger / Confirmer 目前不在前端单独配置

## 三、V2 选股结果怎么看

执行 `POST /api/v1/strategy/run-v2` 后，每只股票结果里最重要的字段如下：

| 字段 | 含义 | 怎么看 |
|---|---|---|
| `matched_strategies` | 实际命中的 Trigger 名称列表 | 看这只股票是靠哪些 Trigger 进来的 |
| `match_count` | 命中的 Trigger 数量 | 多不一定更好，最终还是看 `final_score` |
| `quality_score` | Layer 1 质量分，0-100 | 越高说明基本面越扎实 |
| `tags` | 风格标签及强度 | 如 `growth: 0.82`、`dividend: 0.67` |
| `triggered_signals` | 每个 Trigger 的信号组、置信度、静态权重 | 用来看“为什么它排前面” |
| `confirmed_bonus` | Confirmer 叠加后的总加分 | 当前封顶 `0.6` |
| `dynamic_weight` | 该股票已触发 Trigger 的平均滚动绩效乘数 | 展示字段，不是单独加分项 |
| `style_bonus` | 风格增益 | 由成长/红利标签与市场状态共同决定 |
| `market_regime` | 当前市场状态 | `bull` / `range` / `bear` |
| `final_score` | 最终排序分数 | 选股排名的核心字段 |

### 一句话理解

- `quality_score` 看“底子”
- `matched_strategies` 看“为什么进池”
- `confirmed_bonus` 看“有没有额外确认”
- `dynamic_weight` 看“最近这类 Trigger 表现好不好”
- `final_score` 才是最终排序结果

## 四、V2 引擎怎么排序

### 1. 当前实际公式

当前 V2 的 Layer 3 公式是：

```text
signal_strength
  = Σ(static_weight × regime_coefficient × rolling_performance × confidence)
  + min(Σ(confirmer_bonus), 0.6)

final_score
  = signal_strength × 0.5
  + quality_score_normalized × 0.4
  + style_bonus × 0.1
```

其中：

- `static_weight = ai_rating / 8.32`
- `quality_score_normalized = quality_score / 100`
- `rolling_performance` 收敛到 `[0.8, 1.2]`
- `style_bonus` 收敛到 `[-0.3, 0.3]`
- `confirmer_bonus` 封顶 `0.6`

### 2. 市场状态怎么影响 Trigger

当前市场状态分为三类：

- `bull`：趋势向上
- `range`：震荡
- `bear`：趋势向下

系统对不同信号组的系数如下：

| 信号组 | 牛市 `bull` | 震荡 `range` | 熊市 `bear` |
|---|---:|---:|---:|
| `aggressive` 进攻组 | 1.2 | 0.8 | 0.5 |
| `trend` 趋势组 | 1.0 | 0.9 | 0.6 |
| `bottom` 底部组 | 0.7 | 1.0 | 1.2 |

可以直接这么理解：

- 牛市更偏爱 **进攻组**
- 震荡市更偏爱 **趋势组和底部组**
- 熊市明显更偏爱 **底部组**

### 3. 风格增益怎么影响排序

系统目前只识别两种风格标签：

- `growth`：成长
- `dividend`：红利

对应的市场风格偏好如下：

| 风格 | 牛市 `bull` | 震荡 `range` | 熊市 `bear` |
|---|---:|---:|---:|
| `growth` | 1.2 | 0.9 | 0.7 |
| `dividend` | 0.8 | 1.1 | 1.3 |

换句话说：

- 牛市会更偏爱成长风格
- 熊市会更偏爱高股息风格
- 震荡市偏好更均衡

### 4. 为什么 `dynamic_weight` 显示了，但没有单独加到总分里

因为它现在的正确口径是：

- **先乘进 Trigger 有效权重**
- 再通过 `signal_strength` 进入最终分数

所以你在结果里看到的 `dynamic_weight`，主要是解释用字段，帮助你理解：

- 这只股票依赖的 Trigger，最近 30 天表现偏强还是偏弱

### 1. 总览表

| 英文名 | 中文名 | 分组 | AI 评分 | 适合行情 | 核心用途 |
|---|---|---|---:|---|---|
| `dragon-turnaround-trigger-v2` | 龙回头 | 进攻组 | 8.32 | 牛市强趋势 | 做主升浪二次启动 |
| `first-negative-reversal-trigger-v2` | 首阴反包 | 进攻组 | 7.73 | 强势接力 | 做弱转强反包 |
| `volume-breakout-trigger-v2` | 放量突破 | 进攻组 | 7.42 | 牛市/题材活跃 | 做平台突破 |
| `volume-surge-continuation-trigger-v2` | 后量超前量 | 进攻组 | 7.13 | 趋势加速 | 做量能持续强化 |
| `volume-contraction-pullback-trigger-v2` | 缩量回调 | 趋势组 | 7.42 | 牛市/震荡偏强 | 做趋势中的低吸 |
| `peak-pullback-stabilization-trigger-v2` | 高位回落企稳 | 趋势组 | 7.85 | 龙头二波 | 做主升后的二次启动 |
| `pullback-half-rule-trigger-v2` | 回调半分位 | 趋势组 | 7.00 | 强趋势 | 做浅回调延续 |
| `atr-breakout-trigger-v2` | ATR 波动率突破 | 趋势组 | 6.97 | 横盘后突破 | 做波动率扩张 |
| `extreme-shrink-bottom-trigger-v2` | 地量见底 | 底部组 | 6.68 | 熊市/超跌区 | 做底部试错 |
| `volume-price-stable-trigger-v2` | 量缩价稳 | 底部组 | 6.68 | 熊市/震荡末端 | 做企稳确认 |

### 2. 进攻组（Aggressive）

#### `dragon-turnaround-trigger-v2`｜龙回头

- **定位**：V2 里最强的进攻型 Trigger，适合主线强股的二次启动
- **AI 评分**：`8.32`
- **核心逻辑**：
  - 先有一次强势放量突破（T0）
  - 之后经历 3-8 天左右的洗盘
  - 洗盘期间量能明显缩下来
  - 回踩后重新企稳，准备再起一波
- **默认参数**：
  - `min_t0_pct_chg=6.0`：突破日涨幅至少 6%
  - `min_t0_vol_ratio=2.5`：突破日量比至少 2.5
  - `min_washout_days=3`：洗盘最少 3 天
  - `max_washout_days=8`：洗盘最多 8 天
  - `max_vol_shrink_ratio=0.4`：洗盘量能收缩到突破日的 40% 左右
- **适合什么时候用**：
  - 牛市
  - 题材主线清晰的时候
  - 龙头股从第一次突破后回踩不死时
- **什么时候慎用**：
  - 熊市
  - 纯超跌反抽
  - 板块无持续性的杂毛股
- **搭配建议**：
  - 最适合搭配 `高位回落企稳`、`放量突破`
  - 如果股票还带 `growth` 标签，牛市里会更占优

#### `first-negative-reversal-trigger-v2`｜首阴反包

- **定位**：强势股弱转强接力信号
- **AI 评分**：`7.73`
- **核心逻辑**：
  - 前一日先出现收阴
  - 当天重新走强，阳线反包
  - 当前仍处于上升趋势，不是下跌途中的假反弹
  - 量能至少不弱
- **默认参数**：
  - `min_pct_chg=2.0`：反包日涨幅至少 2%
  - `min_vol_ratio=1.0`：反包日量能至少不低于常态
- **适合什么时候用**：
  - 强势股第一次分歧后的修复
  - 短线情绪修复期
- **什么时候慎用**：
  - 连续退潮阶段
  - 跌破核心均线后的机械反抽
- **搭配建议**：
  - 与 `龙回头`、`后量超前量`一起选时，适合抓强势延续

#### `volume-breakout-trigger-v2`｜放量突破

- **定位**：最通用的突破型 Trigger
- **AI 评分**：`7.42`
- **核心逻辑**：
  - 股价创近期新高
  - 同时量比显著放大
  - 排除停牌和无成交异常
- **默认参数**：
  - `high_period=20`：看 20 日高点
  - `min_vol_ratio=2.0`：量比至少 2.0
- **适合什么时候用**：
  - 牛市
  - 热点板块共振突破
  - 平台整理后放量向上
- **什么时候慎用**：
  - 震荡市里追突破，容易被假突破骗线
  - 成交量只是盘中脉冲，不是有效放量
- **搭配建议**：
  - 想要更稳，就和 `缩量回调`、`ATR 波动率突破`一起用

#### `volume-surge-continuation-trigger-v2`｜后量超前量

- **定位**：趋势加速信号，强调“量能持续强化”
- **AI 评分**：`7.13`
- **核心逻辑**：
  - 当日量比明显放大
  - 短期均量继续强于中期均量
  - 股价有正涨幅
  - 本身已经处于向上趋势里
- **默认参数**：
  - `surge_ratio=2.0`：量比至少 2.0
  - `vol_ma_ratio=1.2`：短期均量相对中期均量至少高 20%
  - `min_pct_chg=1.0`：涨幅至少 1%
- **适合什么时候用**：
  - 主升浪中段
  - 板块强度继续抬升时
- **什么时候慎用**：
  - 量能末端冲刺
  - 见顶前最后一棒
- **搭配建议**：
  - 和 `放量突破`搭配适合做“刚突破 + 持续放量”

### 3. 趋势组（Trend）

#### `volume-contraction-pullback-trigger-v2`｜缩量回调

- **定位**：趋势低吸里最好用的一类
- **AI 评分**：`7.42`
- **核心逻辑**：
  - MA5 在 MA20 上方，说明主趋势没坏
  - 股价回调到 MA20 附近
  - 回调过程中量能缩小
- **默认参数**：
  - `max_vol_ratio=0.6`：回调时量比不超过 0.6
  - `ma_tolerance=0.02`：股价距离 MA20 的容差约 2%
- **适合什么时候用**：
  - 牛市或偏强震荡市
  - 有主趋势但不想追高的时候
- **什么时候慎用**：
  - MA20 已经走平或走坏
  - 回调本质上是趋势反转
- **搭配建议**：
  - 和 `均线多头排列（确认）`、`缩量上涨（确认）`的组合最自然

#### `peak-pullback-stabilization-trigger-v2`｜高位回落企稳

- **定位**：龙头二波 / 主升后再启动的核心信号
- **AI 评分**：`7.85`
- **核心逻辑**：
  - 前面先有一段主升浪
  - 从高位回落一定幅度，但不是彻底走坏
  - 回落后量能缩下来，价格在 MA5 附近企稳
  - 当天再放量走出小阳启动
- **默认参数**：
  - `min_peak_rise_pct=20.0`：前期高点至少相对 MA60 高出 20%
  - `min_pullback_pct=10.0`：至少有一定幅度回落
  - `max_pullback_pct=35.0`：回落也不能太深
  - `max_vol_ratio=0.8`：企稳阶段不能放大量
  - `min_signal_vol_ratio=1.2`：启动日要重新放量
- **适合什么时候用**：
  - 龙头回调后的二次起爆
  - 牛市主线反复强化阶段
- **什么时候慎用**：
  - 退潮中的“高位补跌”
  - 反抽而不是二波
- **搭配建议**：
  - 最适合与 `龙回头`一起使用

#### `pullback-half-rule-trigger-v2`｜回调半分位

- **定位**：强趋势中的浅回调延续信号
- **AI 评分**：`7.00`
- **核心逻辑**：
  - MA5 > MA20 > MA60，多头排列
  - 当天是小幅回调，而不是大阴线
  - 收盘仍在 MA20 上方，也没有跌破 MA5 与 MA20 的中间位置
  - 回调时量能偏小
- **默认参数**：
  - `max_pullback_pct=3.0`：回调不能太深
  - `max_vol_ratio=0.8`：回调最好缩量
- **适合什么时候用**：
  - 已经走出清晰趋势的抱团股
  - 不想等深回调，只做强势趋势延续
- **什么时候慎用**：
  - 高波动题材股
  - 已经临近趋势末端
- **搭配建议**：
  - 适合搭配 `ATR 波动率突破`，前者偏低吸，后者偏突破确认

#### `atr-breakout-trigger-v2`｜ATR 波动率突破

- **定位**：横盘后放大波动的趋势突破信号
- **AI 评分**：`6.97`
- **核心逻辑**：
  - 当日收盘突破 `MA20 + ATR14 × 倍数`
  - 且前一日尚未突破，避免连续重复命中
- **默认参数**：
  - `atr_multiplier=1.5`
- **适合什么时候用**：
  - 平台整理后的第一次异动
  - 趋势股重新放大波动
- **什么时候慎用**：
  - 极端高波动日
  - 一字板、情绪板、纯消息刺激
- **搭配建议**：
  - 和 `放量突破`一起时，信号会更偏“有效突破”

### 4. 底部组（Bottom）

#### `extreme-shrink-bottom-trigger-v2`｜地量见底

- **定位**：左侧试错型底部信号
- **AI 评分**：`6.68`
- **核心逻辑**：
  - 成交极度萎缩
  - 换手率极低
  - 不是一字板，也不是跌停
- **默认参数**：
  - `extreme_ratio=0.3`：量比极低
  - `max_turnover=1.0`：换手率上限 1%
- **适合什么时候用**：
  - 熊市末端
  - 长时间阴跌后的底部观察
- **什么时候慎用**：
  - 还在主跌浪中途
  - 流动性很差的小票
- **搭配建议**：
  - 最好搭配 `RSI 超卖（确认）`、`MACD 底背离（确认）`

#### `volume-price-stable-trigger-v2`｜量缩价稳

- **定位**：比“地量见底”更偏确认，不那么激进
- **AI 评分**：`6.68`
- **核心逻辑**：
  - 量能缩小
  - 涨跌幅很小，价格趋于稳定
  - 股价靠近 MA20 附近
  - 不是跌停崩盘日
- **默认参数**：
  - `max_vol_ratio=0.5`
  - `max_pct_chg=2.0`
  - `ma_position=1.02`
- **适合什么时候用**：
  - 震荡末端
  - 底部筑平台阶段
- **什么时候慎用**：
  - 还没真正止跌
  - 突发利空刚释放完的阶段
- **搭配建议**：
  - 和 `地量见底`一起使用，比单独做左侧更稳一点

## 六、系统自动参与的内部策略

> 这些策略不会出现在工作台“勾选列表”里，但会自动参与排序。  
> 真正理解 V2 排名，必须知道它们在做什么。

### 1. Guard（排雷）策略

Guard 是 **AND 逻辑**：

- 任意一个 Guard 不通过，这只股票就直接被剔除

#### `financial-safety-guard-v2`｜财务安全（排雷）

- **作用**：排除高负债、流动性不足的公司
- **默认规则**：
  - `debt_ratio < 60`
  - `current_ratio >= 1.5`
- **适合的理解方式**：
  - 这是最基础的财务安全门槛
  - 它不是找“最好”的公司，而是先排掉明显危险的公司

#### `cashflow-quality-guard-v2`｜现金流质量（排雷）

- **作用**：排除“利润好看但现金流差”的公司
- **默认规则**：
  - `EPS > 0`
  - `OCF per share > 0`
  - `OCF / EPS >= 1.0`
- **适合的理解方式**：
  - 它专门防纸面利润
  - 对成长股尤其重要

### 2. Scorer（质量评分）策略

#### `quality-score-v2`｜综合质量评分

- **作用**：给 Layer 1 股票打一个 `0-100` 的连续分数
- **AI 评分**：`7.80`
- **当前权重**：

| 维度 | 权重 | 说明 |
|---|---:|---|
| ROE | 25% | 盈利能力 |
| 利润同比增速 | 20% | 成长性 |
| 资产负债率 | 20% | 安全性 |
| PE TTM | 20% | 估值 |
| 毛利率 | 15% | 业务质量 |

- **重要说明**：
  - 关键数据缺失时，分数会被拉到很低甚至 `0`
  - `quality_score` 高，不代表短线马上涨
  - 它的意义是：让同样触发信号的股票里，**底子更好的排前面**

### 3. Tagger（风格标签）策略

Tagger 不负责“能不能入选”，而负责：

- 给股票打风格标签
- 输出强度值 `0.0-1.0`
- 在 Layer 3 里生成 `style_bonus`

#### `low-pe-high-roe-tagger-v2`｜低估值高成长（标签）

- **风格标签**：`growth`
- **AI 评分**：`7.62`
- **默认规则**：
  - `PE < 30`
  - `ROE >= 15%`
  - `profit_yoy >= 20%`
- **风格强度来源**：
  - PE 越低越好
  - ROE 越高越好
  - 利润增速越高越好
- **什么时候更有优势**：
  - 牛市
  - 主升浪环境

#### `high-dividend-tagger-v2`｜高股息（标签）

- **风格标签**：`dividend`
- **AI 评分**：`8.08`
- **默认规则**：
  - `dividend_yield >= 3%`
  - `PE < 20`
- **风格强度来源**：
  - 股息率越高越好
  - PE 越低越好
- **什么时候更有优势**：
  - 熊市
  - 震荡市
  - 防御行情

### 4. Confirmer（辅助确认）策略

Confirmer 不会独立触发股票，只会给已经触发的股票加分。

#### 总体规则

- 加分是**加法叠加**
- 总加分**封顶 `0.6`**
- 每个 Confirmer 只对特定信号组生效

#### 5 个 Confirmer 一览

| 策略 | 默认加分 | 适用信号组 | 作用 |
|---|---:|---|---|
| `ma-long-arrange-confirmer-v2` | 0.3 | `trend` | 确认趋势方向没坏 |
| `rsi-oversold-confirmer-v2` | 0.2 | `bottom` | 确认超跌反弹环境 |
| `bias-extreme-confirmer-v2` | 0.2 | `bottom` | 确认价格乖离极端超卖 |
| `macd-divergence-confirmer-v2` | 0.3 | `bottom` | 确认底背离 |
| `shrink-volume-rise-confirmer-v2` | 0.2 | `trend` | 确认筹码锁定、趋势延续 |

#### `ma-long-arrange-confirmer-v2`｜均线多头排列（确认）

- **核心逻辑**：`MA5 > MA10 > MA20 > MA60`
- **用途**：给趋势组信号加 `0.3`
- **你应该怎么理解**：这类确认的意义是“别在趋势坏掉的时候做趋势低吸”

#### `rsi-oversold-confirmer-v2`｜RSI 超卖（确认）

- **默认参数**：
  - `period=6`
  - `oversold=20`
  - `bounce=30`
- **用途**：给底部组加 `0.2`
- **你应该怎么理解**：底部组不是越左越好，要有一点超跌反弹的技术面依据

#### `bias-extreme-confirmer-v2`｜BIAS 极端乖离（确认）

- **默认参数**：
  - `oversold_bias=-6.0`
- **用途**：给底部组加 `0.2`
- **你应该怎么理解**：价格已经离均线很远，容易出现均值回归

#### `macd-divergence-confirmer-v2`｜MACD 底背离（确认）

- **默认参数**：
  - `lookback=20`
- **用途**：给底部组加 `0.3`
- **你应该怎么理解**：它不是找“跌得多”，而是找“跌不动了”

#### `shrink-volume-rise-confirmer-v2`｜缩量上涨（确认）

- **默认参数**：
  - `max_vol_ratio=0.8`
  - `min_pct_chg=0.5`
- **用途**：给趋势组加 `0.2`
- **你应该怎么理解**：上涨不放巨量，通常意味着筹码更稳

## 七、独立保留的 V4 策略

### `volume-price-pattern`｜量价配合（龙回头）

- **定位**：独立于 V2 主流程的遗留活跃策略
- **特点**：
  - 不是简单的当天布尔筛选
  - 带有观察池、T0/Tk 两阶段逻辑
  - 支持大盘过滤、行业强度过滤、观察池过期机制
- **典型默认参数**：
  - `accumulation_days=60`
  - `min_t0_pct_chg=6.0`
  - `min_t0_vol_ratio=2.5`
  - `min_washout_days=3`
  - `max_washout_days=8`
  - `max_vol_shrink_ratio=0.4`
  - `market_filter_enabled=True`
  - `sector_filter_enabled=True`
- **适合谁用**：
  - 想单独研究龙回头两阶段演化的人
  - 想要更细粒度观察池管理的人
- **和 V2 的关系**：
  - V2 里的 `dragon-turnaround-trigger-v2` 是主路径版、轻量版
  - `volume-price-pattern` 是保留下来的独立版本，适合做专题研究

## 八、按使用场景选策略

### 1. 如果你偏短线进攻

推荐优先从进攻组里选：

- `龙回头`
- `首阴反包`
- `放量突破`

建议搭配：

- 牛市：`龙回头 + 放量突破 + 后量超前量`
- 情绪修复：`首阴反包 + 放量突破`

### 2. 如果你偏趋势低吸

推荐优先从趋势组里选：

- `缩量回调`
- `高位回落企稳`
- `回调半分位`

建议搭配：

- 中强趋势：`缩量回调 + 回调半分位`
- 龙头二波：`高位回落企稳 + 龙回头`

### 3. 如果你偏底部试错

推荐优先从底部组里选：

- `地量见底`
- `量缩价稳`

建议搭配：

- 熊市末端：`地量见底 + 量缩价稳`
- 想更稳一点：只开底部组，不开进攻组

### 4. 按市场状态来选

| 市场状态 | 建议优先组别 | 不建议重仓的组别 |
|---|---|---|
| `bull` | 进攻组、趋势组 | 底部组 |
| `range` | 趋势组、底部组 | 纯进攻组 |
| `bear` | 底部组 | 进攻组 |

### 5. 给你一个最实用的经验法则

- **不要第一次就勾满 10 个 Trigger**
- 初次使用建议只选 **2-4 个**
- 如果你不知道怎么选，从下面三套开始：

1. **牛市突破版**：`龙回头 + 放量突破 + 后量超前量`
2. **趋势低吸版**：`缩量回调 + 回调半分位 + ATR 波动率突破`
3. **防守试错版**：`地量见底 + 量缩价稳`

## 九、常用 API

### 1. 获取可选 Trigger 列表

```bash
curl http://localhost:8000/api/v1/strategy/list-v2
```

返回的是**10 个 Trigger**，不是全部 20 个内部策略。

### 2. 执行 V2 选股

```bash
curl -X POST http://localhost:8000/api/v1/strategy/run-v2 \
  -H "Content-Type: application/json" \
  -d '{
    "strategy_names": [
      "dragon-turnaround-trigger-v2",
      "volume-breakout-trigger-v2",
      "volume-contraction-pullback-trigger-v2"
    ],
    "target_date": "2026-03-07",
    "top_n": 30,
    "markets": ["主板", "创业板"]
  }'
```

### 3. 查询/更新 Trigger 配置

```bash
# 查询所有 Trigger 配置
curl http://localhost:8000/api/v1/strategy/config

# 更新单个 Trigger
curl -X PUT http://localhost:8000/api/v1/strategy/config/volume-breakout-trigger-v2 \
  -H "Content-Type: application/json" \
  -d '{
    "params": {
      "high_period": 20,
      "min_vol_ratio": 2.2
    },
    "is_enabled": true
  }'

# 批量更新
curl -X PUT http://localhost:8000/api/v1/strategy/config/batch \
  -H "Content-Type: application/json" \
  -d '{
    "strategies": [
      {
        "name": "volume-breakout-trigger-v2",
        "params": {"min_vol_ratio": 2.2},
        "is_enabled": true
      },
      {
        "name": "extreme-shrink-bottom-trigger-v2",
        "params": {"extreme_ratio": 0.25},
        "is_enabled": false
      }
    ]
  }'
```

### 4. 获取行业和市场筛选项

```bash
curl http://localhost:8000/api/v1/strategy/industries
curl http://localhost:8000/api/v1/strategy/markets
```

### 5. 回测

```bash
curl -X POST http://localhost:8000/api/v1/backtest/run \
  -H "Content-Type: application/json" \
  -d '{
    "strategy_name": "volume-breakout-trigger-v2",
    "stock_codes": ["600519.SH", "000858.SZ"],
    "start_date": "2024-01-01",
    "end_date": "2025-12-31",
    "initial_capital": 1000000
  }'
```

> 回测接口当前按**单策略**运行，更适合检验单个 Trigger 或独立策略。

## 十、常见问题

### 1. 为什么我勾了很多策略，但结果反而更乱？

因为多 Trigger 不是简单“多多益善”。  
你勾得越多，候选池越大，风格和行情类型也更杂。建议：

- 先按你的交易风格选 2-4 个
- 再逐步增加，而不是第一次就开满

### 2. 为什么有的股票 `match_count` 少，但 `final_score` 更高？

因为最终排名不是按命中个数，而是按：

- Trigger 静态权重
- 市场状态系数
- 滚动绩效
- 质量分
- 确认加分
- 风格增益

综合算出来的。

### 3. 为什么 `market_regime=bear` 时，进攻组结果明显减少？

这是系统故意这样设计的。  
进攻组在熊市的系数只有 `0.5`，底部组则是 `1.2`。也就是说，熊市里系统会主动压低追涨型信号的排序权重。

### 4. 为什么策略配置页里只有 10 个策略？

因为配置页面对的是**用户主动可控部分**，也就是 Trigger。  
Guard / Scorer / Tagger / Confirmer 是系统内部自动运行的，不在当前页面单独暴露。

### 5. `dynamic_weight` 到底是不是加分项？

不是单独加分项。  
它现在是**并入 Trigger 权重的乘数**，只是结果里额外展示出来，方便你理解这个 Trigger 最近 30 天表现偏强还是偏弱。

### 6. 当前 `run-v2` 会不会再做 AI 终审？

当前工作台主链路的核心排序来自 **V2 多因子融合**。  
`ai_enabled` 字段会返回，但不应该把它当成当前排序的主要来源。日常盘后 AI 分析是另一条独立链路。

## 最后的建议

如果你只想要一个最简上手方案，直接照这个来：

1. 第一次先试 `龙回头 + 放量突破 + 缩量回调`
2. 看结果里的 `market_regime` 和 `quality_score`
3. 如果结果太激进，去掉进攻组，换成 `缩量回调 + 回调半分位`
4. 如果是熊市，优先只看 `地量见底 + 量缩价稳`
5. 当某个 Trigger 经常太宽或太窄，再去配置页微调参数

这套流程最接近当前系统的真实使用方式。
